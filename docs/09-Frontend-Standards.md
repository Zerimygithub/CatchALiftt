# 🎨 CatchALiftt Frontend Standards

---

# Purpose

This document defines the frontend development standards for the CatchALiftt platform.

Following these standards ensures:

- Consistent UI
- Reusable code
- Easy maintenance
- Better performance
- Professional project structure

# CSS Rules

Use one CSS file per page.

Reusable styles belong in

style.css

Avoid inline CSS.

Avoid !important unless absolutely necessary.

# Buttons

Primary Button

Gold Background

White Text

Rounded Corners

---

# Icons

Use SVG icons whenever possible.

Avoid PNG icons.

---

# Images

Compress before upload.

Use WebP when possible.

Lazy load large images.

---

# Responsive Design

Mobile First

Breakpoints

480px

768px

1024px

1440px

---

# JavaScript Rules

One JS file per page.

Avoid global variables.

Use modules whenever possible.

Separate UI logic from business logic.

---

# Performance

Minimize DOM manipulation.

Use requestAnimationFrame for animations.

Debounce search.

Lazy load assets.

---

# Accessibility

Semantic HTML

Keyboard navigation

ARIA labels where needed

Visible focus indicators

Sufficient color contrast

---
