# 📘 CatchALiftt Business Rules

---

# Purpose

This document defines the operational rules that govern the CatchALiftt platform.

Business Rules ensure that every user, ride, payment, and AI decision behaves consistently across the application.

---

# User Rules

A user may act as both:

- Traveller
- Host

using a single account.

No duplicate accounts are allowed using the same mobile number.

---

# Registration Rules

Required

- First Name
- Mobile Number

Optional

- Email
- Profile Photo

Verification required before becoming a Host.

---

# Host Rules

Host must:

- Have a verified mobile number
- Complete profile
- Register vehicle
- Accept platform policies

Host may create multiple rides.

Host may cancel rides before departure.

---

# Traveller Rules

Traveller can:

- Search rides
- Book seats
- Cancel booking
- Rate Host

Traveller cannot exceed available seats.

---

# Ride Rules

Ride must include:

- Pickup
- Destination
- Date
- Time
- Available Seats
- Price

Completed rides cannot be edited.

Cancelled rides cannot accept bookings.

---

# Booking Rules

Booking becomes confirmed only after Host approval.

Booking expires automatically if Host does not respond within the configured time.

---

# Cancellation Rules

Traveller

- More than 60 minutes before pickup
  No penalty

- 30–60 minutes
  Small penalty

- Less than 30 minutes
  Higher penalty

- No Show
  Maximum penalty

Host

Repeated cancellations reduce Host reliability score.

---

# Wallet Rules

Traveller pays through Wallet.

Platform deducts payment only after ride completion.

Refunds are processed automatically when applicable.

Wallet balance cannot become negative.

---

# Payment Rules

Platform commission is deducted automatically.

Host receives remaining amount.

Every transaction receives a unique Transaction ID.

---

# Rating Rules

Traveller rates Host.

Host rates Traveller.

Ratings are allowed only after ride completion.

Ratings cannot be edited after submission.

---

# AI Rules

AI recommends:

- Smart Pickup
- Route Optimization
- Ride Matching
- Demand Prediction

AI must not increase Host travel time beyond configured limits.

---

# Notification Rules

Notify users for:

- Booking request
- Booking accepted
- Booking rejected
- Ride reminder
- Ride completed
- Wallet updates

---

# Safety Rules

SOS available during active rides.

Emergency contacts can be notified.

Reported users may be suspended after review.

---

# Referral Rules

Referral reward is granted only after the referred user completes the first successful ride.

---

# Reward Rules

Users earn reward points for:

- Completing rides
- Hosting rides
- Referrals
- High ratings

---

# Admin Rules

Admin can:

- Suspend accounts
- Resolve disputes
- Review SOS cases
- Generate reports
- Manage rewards

---

# Future Rules

- Corporate rides
- Subscription plans
- Carbon credits
- AI dynamic pricing
- Fleet operators
